Stop hand‑editing config.

visual-config turns your JavaScript or TypeScript project's configuration into one clear interface: dependencies, scripts, tsconfig, linters, formatters. Every change becomes a diff you approve before it touches a file, and anything applied can be undone. Your AI agents get the same safe controls over MCP.

It's one command in any JS/TS project. Nothing to install or configure:

$ npx @apostel/visual-config
See what it does ↓

What you can do with it today.

Version 0.7 is a young alpha, but all of this works right now, on your real project.

Catch unhealthy dependencies before they bite

Vulnerabilities, deprecations, and outdated versions are pulled straight from the npm registry and shown as facts on each package.

  • Changelog shows the release notes between your version and latest, breaking changes highlighted
  • Safe? cross-references those breaking changes against how your code uses the package
  • Upgrade all bundles every outdated package into a single reviewed diff

Edit configs as guided forms, not raw JSON

Biome, Prettier, ESLint, oxlint, and tsconfig open as forms with each option documented inline. Edits preserve your file's formatting and comments.

  • next.config, vite.config, and flat ESLint configs get read-only views, statically extracted from the code
  • Set up Prettier, Biome, or oxlint from scratch in one step
  • One-click Switch to Biome replaces ESLint + Prettier, fully reversible

Make the daily chores harder to get wrong

The routine work that usually happens in a terminal, with room for typos, becomes clicking on verified data.

  • Install packages by selecting them from live registry search, so a typo can't execute an arbitrary package
  • Scripts run as buttons with streamed output and a Stop control
  • Monorepo aware: a package switcher points every view at any workspace member
  • Every applied change sits in History with an Undo button

Give your AI agents the same guardrails

Every operation in the UI is also an MCP tool. Agents propose the same diffs, write through the same engine, and land in the same undo history, instead of free-typing shell commands at your config.

  • Read-only resources expose the project model and diagnostics
  • MCP Apps support renders an interactive panel inside the agent session
  • init-mcp registers the server for Claude, Cursor, or VS Code in one step 
# the agent-facing server (stdio)
npx @apostel/visual-config mcp

Run it on your project.

Requires Node 20 or newer. Nothing else.

Open the UI

$ npx @apostel/visual-config

Starts a local daemon in the current project and opens the interface in your browser. Local only; no account, no telemetry.

Connect your agent

$ npx @apostel/visual-config init-mcp

Registers the MCP server for Claude, Cursor, and VS Code, so agents in this repo plan config changes as reviewable diffs.

Or just peek at any public repo

$ npx @apostel/visual-config try sindresorhus/got

Clones a repo read-only, reports its outdated, vulnerable, and deprecated dependencies, and prints an upgrade patch to stdout. Never runs the repo's code.

Built to be extended, not owned.

The built-in features are plugins with no special privileges, so anything they can do, your package can do too.

Tool and framework authors

Ship a plugin that gives your ecosystem a first-class panel inside every user's project.

  • definePlugin from @apostel/visual-config-kit
  • Operations you register appear in the UI and as MCP tools automatically
  • Plugins never touch the filesystem; they return Changes the engine reviews

People with opinions worth installing

The base tool states only verifiable facts. Recommendations come from opinion packs users choose, always labeled with the author's name.

  • "Enable strict" appears as your advice, attributed to you
  • Users stack packs and see where they disagree
  • Verified and community badges keep provenance visible

The rules it can't break.

These are enforced by the code, not aspirations.

Your files stay the only source of truth

No shadow store, no lock-in. It reads and writes the files git already tracks. Uninstalling leaves your project byte-for-byte as it was.

Nothing writes without a confirmed diff

Planning a change never touches disk. One engine method applies changes, and each operation declares which files it may write; the engine rejects the rest.

Formatting and comments survive every edit

Edits are the minimal diff a careful human would make. The write layer is golden-tested, because the first clobbered comment is the end of trust.

Everything is undoable

Applied changes land in a journal with their inverse. Undo works the same whether a human or an agent made the change.

Facts and opinions never mix

The base reports only what's verifiable: vulnerable, outdated, deprecated, non-default. Advice comes from opinion packs you install, with the author's name on it.

Local-first, no account, no telemetry

It runs on your machine against your files. That's the entire data flow.
